本地 LLM + RAG 系统开发计划

一、项目概述

构建一个完全本地化部署的 大语言模型（LLM）+ 检索增强生成（RAG） 系统，实现基于私有知识库的智能问答、文档检索与内容生成能力。所有数据和模型均在本地运行，确保数据隐私与安全。

四、开发阶段与里程碑

第一阶段：环境搭建与基础验证（第1-2周）

ollama pull llama3:70b            # 如显存充足
ollama pull nomic-embed-text      # Embedding模型
  • 初始化 Python 项目结构

1.2 基础功能验证

  • 验证 Ollama 本地 LLM 调用（REST API / Python SDK）
  • 验证 Embedding 模型生成向量
  • 验证向量数据库基本 CRUD 操作
  • 完成一个最简 RAG 问答 demo（端到端跑通）

里程碑：能通过命令行输入问题，从测试文档中检索并生成回答。

第二阶段：文档处理管线（第3-4周）

2.1 文档加载与解析

  • 支持多格式文档加载
    • PDF（含扫描版OCR）
    • Word (.docx)
    • Markdown
    • HTML / 网页
    • TXT / CSV
    • PowerPoint (.pptx)
  • 文档元数据提取（标题、作者、日期、来源等）
  • 表格与图片内容提取

2.2 文本分块策略（Chunking）

  • 实现多种分块策略
    • 固定大小分块：按字符数/Token数切分
    • 递归字符分块：按分隔符递归切分（推荐默认）
    • 语义分块：基于语义相似度切分
    • 文档结构分块：基于标题/章节切分
  • 分块参数可配置（chunk_size, chunk_overlap）
  • 分块质量评估工具

2.3 向量化与入库

  • 批量文档向量化处理
  • 增量更新机制（新增/修改/删除文档）
  • 向量化进度跟踪与日志
  • 文档去重检测

里程碑：支持上传多种格式文档，自动处理并入库，支持增量更新。

第三阶段：检索与生成优化（第5-7周）

3.1 检索策略优化

  • 基础向量检索：余弦相似度 / L2距离
  • 混合检索（Hybrid Search）：向量检索 + BM25关键词检索
  • 多路召回：不同检索策略结果融合
  • Reranker重排序：使用 bge-reranker 对初检结果精排
  • 元数据过滤：按文档来源、日期、类型等过滤
  • 检索参数调优（top_k, score_threshold 等）

3.2 Prompt工程

  • 设计系统 Prompt 模板
  • 上下文注入策略优化
  • 多轮对话 Prompt 管理
  • Prompt 模板版本管理
  • 支持不同场景的 Prompt 模板（问答/摘要/分析等）

3.3 生成质量优化

  • 引用来源标注（生成结果标注引用的文档片段）
  • 答案置信度评估
  • 幻觉检测与缓解策略
  • 长文档摘要生成
  • 流式输出（Streaming）

3.4 查询优化

  • 查询改写（Query Rewriting）：优化用户输入
  • 查询扩展（Query Expansion）：自动扩展关键词
  • HyDE（Hypothetical Document Embedding）：假设性文档嵌入
  • 多步检索（Multi-step Retrieval）：复杂问题分步检索

里程碑：检索准确率和生成质量显著提升，支持引用来源标注。

第四阶段：API服务与前端开发（第8-9周）

4.1 后端 API 服务

  • 基于 FastAPI 构建 RESTful API

    • POST /api/documents/upload — 文档上传
    • GET /api/documents — 文档列表
    • DELETE /api/documents/{id} — 删除文档
    • POST /api/knowledge-base/create — 创建知识库
    • GET /api/knowledge-base/list — 知识库列表
    • POST /api/search — 独立检索接口
    • GET /api/system/status — 系统状态
  • API 认证与权限管理
  • 请求速率限制
  • API 文档自动生成（Swagger/OpenAPI）
  • WebSocket 支持（实时流式输出）

4.2 前端界面

  • 对话界面（类ChatGPT风格）
    • 多轮对话支持
    • 流式打字效果
    • 引用来源展示（可点击跳转）
    • 对话历史管理
  • 知识库管理界面
    • 文档上传（拖拽上传）
    • 文档列表与状态展示
    • 知识库创建/删除/配置
  • 系统配置界面
    • 模型选择与参数调整
    • 检索参数配置

  • 系统监控仪表板

里程碑：完成可用的Web界面，支持文档管理和知识库问答。

七、关键技术风险与应对

风险 影响 应对策略
GPU显存不足 无法加载大模型 使用量化模型（Q4_K_M）；选用小参数模型；CPU推理兜底
检索精度不够 回答质量差 混合检索 + Reranker；优化分块策略；增加元数据过滤
中文处理效果差 分词/理解不准 选择英文优化模型（Llama3/Mistral）；使用英文Embedding模型
文档解析失败 部分文档无法入库 多解析器兜底；人工标注兜底；OCR增强
幻觉问题 生成不准确内容 严格Prompt约束；引用来源核验；置信度评估过滤
并发性能瓶颈 响应慢/超时 请求队列；模型推理缓存；异步处理
